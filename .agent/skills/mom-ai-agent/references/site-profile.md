## Content Direction

The site already points toward a knowledge-hub model:

- FAQ content with product and trust questions
- Blog content across safe sleep, newborn feeding, postpartum depression, fever, teething, babyproofing, and developmental play
- GEO-oriented content architecture in the repository README and Next.js project README

## Writing Alignment

When generating MomAI-aligned content:

- Keep the tone calm, supportive, and practical
- Favor answer-first educational content
- Tie content to parenting stages and caregiver decision-making
- Use product references only when they fit the public site narrative
- Avoid claiming private clinical validation, hidden datasets, or unpublished APIs unless the user explicitly provides that information

## Good Output Targets

- Blog post summaries or drafts for `momaiagent.com/blog`
- FAQ entries for the site FAQ page
- Product education copy for DearBaby or SolidStart
- Knowledge bundles that can be ingested into the MomAI content engine
